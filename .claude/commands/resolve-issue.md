# GitHub 이슈 해결 계획

당신은 소프트웨어 엔지니어링 전문가입니다.

GitHub 이슈 #$ARGUMENT를 조회하고 해결 계획을 수립하세요.

---

## 🔍 Step 1: 이슈 조회

다음 명령어로 이슈 정보를 조회합니다:

```bash
gh issue view $ARGUMENT --json title,body,labels,state,assignees,createdAt
```

조회 결과:
- **이슈 제목**
- **이슈 설명**
- **라벨** (bug, feature, documentation 등)
- **상태** (open, closed)
- **담당자**
- **생성일**

---

## 📊 Step 2: 이슈 분석

조회한 이슈에 대해 다음을 분석합니다:

### 2.1 문제 정의
- 이슈가 요청하는 것이 무엇인가?
- 핵심 요구사항은?
- 영향도는?

### 2.2 현황 분석
- 현재 코드베이스에서 관련된 부분?
- 이미 구현된 기능?
- 누락된 부분?

### 2.3 복잡도 평가
- 기술적 난이도: 🟢 낮음 / 🟡 중간 / 🔴 높음
- 예상 소요시간: ?시간
- 위험도: ?

---

## 🏗️ Step 3: 해결 계획 수립

### 3.1 접근 방식 (AS-IS → TO-BE)
```
현재 상태 (AS-IS)
    ↓
필요한 변경사항
    ↓
목표 상태 (TO-BE)
```

### 3.2 작업 분해
각 작업을 다음 형식으로 정의합니다:

```
## 작업 1: [작업명]
- **설명**: [구체적 설명]
- **파일**: [수정할 파일]
- **예상시간**: [시간]
- **의존성**: [선행 작업]
- **체크리스트**:
  - [ ] 세부 작업 1
  - [ ] 세부 작업 2

## 작업 2: [작업명]
...
```

### 3.3 의존성 맵
```
작업 1 (필수)
    ↓
작업 2, 작업 3 (병렬 가능)
    ↓
작업 4 (통합 & 테스트)
```

### 3.4 테스트 전략
- 단위 테스트 (무엇을?)
- 통합 테스트 (무엇을?)
- 수동 테스트 (무엇을?)

### 3.5 위험 & 완화 전략
| 위험 | 영향도 | 완화책 |
|------|--------|--------|
| ? | 높음/중/낮 | ? |

---

## ✅ Step 4: 사용자 검사 요청

계획을 정리한 후, 사용자에게 다음을 요청합니다:

### 검사 체크리스트
- [ ] 이슈 이해도 - 계획이 이슈 요구사항을 정확히 반영하는가?
- [ ] 접근 방식 - 제안된 방식이 최선인가?
- [ ] 작업 분해 - 작업이 충분히 세분화되었는가?
- [ ] 시간 추정 - 예상시간이 합리적인가?
- [ ] 위험 요소 - 누락된 위험이 있는가?

### 사용자 피드백 예상 질문
1. **이 접근법에 동의하시나요?** (Yes / No / Revise)
2. **작업 순서를 바꾸거나 추가하실 게 있나요?**
3. **시간 추정에 동의하시나요?**
4. **특별히 주의할 사항이 있나요?**
5. **계획을 수정하고 진행할까요? (Approve to Proceed)**

---

## 🚀 Step 5: 승인 후 실행

사용자 승인 후:

### 5.1 브랜치 생성
```bash
git checkout -b issue-$ARGUMENT-[issue-title]
```

### 5.2 작업 시작
각 작업을 체크리스트로 추적하면서 진행합니다.

### 5.3 커밋 & PR
```bash
git add .
git commit -m "Fix #$ARGUMENT: [커밋 메시지]"
git push origin issue-$ARGUMENT-[issue-title]
```

### 5.4 PR 생성
```bash
gh pr create --title "Fix #$ARGUMENT" --body "이슈 해결 내용"
```

## 🎯 주의사항

- ✅ GitHub CLI (`gh`) 설치 필수
- ✅ 이슈 번호는 정수만 입력 (예: `123`, `#123` 아님)
- ✅ 개인 저장소 또는 권한 있는 저장소에서만 실행
- ✅ 민감한 정보(API 키, 비밀번호) 노출 주의
- ✅ 이슈 내용에 오류나 불명확한 부분이 있으면 지적
