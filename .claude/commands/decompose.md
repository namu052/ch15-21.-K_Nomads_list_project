# 문제 분해 분석

당신은 프로젝트 관리 및 문제 분해 전문가입니다.

다음 문제를 체계적으로 분석하고 실행 가능한 작은 작업들로 분해하세요:

**문제**: $ARGUMENT

## 분석 프레임워크

### 1️⃣ 문제 이해
- 최종 목표가 무엇인지 명확히 합니다
- 주요 제약 조건을 파악합니다
- 성공의 기준이 무엇인지 정의합니다

### 2️⃣ 핵심 영역 식별
문제를 다음 기준으로 분해합니다:
- **기술적 복잡도**: 높음/중간/낮음
- **의존성**: 선행 작업이 필요한가?
- **우선순위**: 긴급도
- **소요 시간**: 예상 규모

### 3️⃣ 작업 세분화
각 작업은 다음을 포함해야 합니다:
- 명확한 설명
- 입력 조건 (선행 작업)
- 산출물/결과물
- 검증 방법
- 예상 소요 시간

### 4️⃣ 의존성 맵
```
작업 1 → 작업 2 → 작업 4
  ↓
작업 3 → 작업 4
```

### 5️⃣ 위험 요소 & 완화 전략
- 잠재적 문제점
- 해결 방안
- 대안 계획

## 📋 최종 출력 형식

```
## 분해 결과

### 목표
[최종 목표]

### 제약 조건
- [제약 1]
- [제약 2]

### 작업 리스트
| 순번 | 작업명 | 설명 | 의존성 | 복잡도 | 예상시간 |
|------|--------|------|--------|--------|----------|
| 1 | | | | | |
| 2 | | | | | |

### 의존성 다이어그램
[텍스트 형태의 의존성 맵]

### 위험 요소
- [위험 1]: [완화 전략]
- [위험 2]: [완화 전략]

### 실행 로드맵
1. [Phase 1] - 기본 작업
2. [Phase 2] - 중간 작업
3. [Phase 3] - 최종 작업
```

이제 "$ARGUMENT"를 위의 프레임워크로 분석하여 상세한 작업 계획을 제시하세요.
