**포트**: http://localhost:3004 (또는 3003, 3002 - 포트 3000이 사용 중이면 자동으로 변경)

## 📱 테스트 URL

### 1. 홈페이지
```
http://localhost:3004/
```
**확인사항**:
- ✅ 페이지 로드됨
- ✅ 도시 카드 목록 보임
- ✅ 7개 도시 카드 표시
- ✅ 필터 버튼 보임
- ✅ 도시 카드에 이모지 표시

### 2. 도시 상세페이지 (7개 모두)
```
http://localhost:3004/cities/부산
http://localhost:3004/cities/대구
http://localhost:3004/cities/대전
http://localhost:3004/cities/전주
http://localhost:3004/cities/강릉
http://localhost:3004/cities/제주
http://localhost:3004/cities/성남
```

**각 페이지에서 확인**:
- ✅ 헤더 고정 (뒤로가기 버튼)
- ✅ 메인 영역 (큰 이모지 + 도시명)
- ✅ 통계 카드 3개 (생활비, 인터넷, 노마드)
- ✅ 필터 배지 4개 (예산, 지역, 환경, 계절)
- ✅ 설명 텍스트
- ✅ 주요 특징 4개
- ✅ 좋아요/싫어요 버튼
- ✅ 추천 도시 섹션
- ✅ 푸터

### 3. 404 페이지 (존재하지 않는 도시)
```
http://localhost:3004/cities/unknown
http://localhost:3004/cities/invalid
```

**확인사항**:
- ✅ 404 페이지 표시됨

## 🧪 테스트 시나리오

### 시나리오 1: 홈페이지 → 부산 상세페이지
1. http://localhost:3004/ 접속
2. 부산 카드 클릭 (🌊 이모지)
3. `/cities/부산` 페이지로 이동 확인
4. 부산 정보 정확하게 표시 확인

### 시나리오 2: 반응형 테스트 (모바일)
1. Chrome DevTools 열기 (F12)
2. 모바일 모드 토글 (Ctrl+Shift+M)
3. iPhone 12 선택 (375x812)
4. 페이지 새로고침
5. 확인:
   - [ ] 텍스트 가독성 좋음
   - [ ] 버튼 크기 적절 (터치 가능)
   - [ ] 이모지 크기 충분
   - [ ] 가로 스크롤 없음
   - [ ] 여백 적절

### 시나리오 3: 좋아요/싫어요 투표
1. 부산 상세페이지 방문
2. 좋아요 버튼 클릭 → 숫자 증가 확인
3. 싫어요 버튼 클릭 → 숫자 증가 확인
4. 페이지 새로고침 (F5)
5. 숫자 유지되는지 확인 (localStorage 작동)
6. 다른 도시 페이지 방문
7. 좋아요/싫어요 값 독립적인지 확인

### 시나리오 4: 네비게이션 검증
1. 부산 상세페이지 접속
2. 뒤로가기 버튼 클릭
3. 홈페이지로 돌아오는지 확인
4. 제주 카드 클릭
5. `/cities/제주`로 이동하는지 확인
6. 제주 정보 표시되는지 확인
7. 추천 도시 중 하나 클릭
8. 해당 도시 페이지로 이동하는지 확인

### 시나리오 5: 데이터 정확도
**부산 검증**:

## 🐛 문제 발생 시

### 포트 충돌
```bash
# 3000 포트가 사용 중이면 자동으로 3001, 3002, 3003, 3004 등으로 변경됨
# 만약 수동으로 변경하고 싶다면:
PORT=3005 npm run dev
```

### 페이지 로드 안 됨
```bash
# 1. 개발 서버 중지 (Ctrl+C)
# 2. node_modules 재설치
npm install

# 3. 캐시 삭제
rm -rf .next

# 4. 다시 시작
npm run dev
```

### 스타일 안 맞음
```bash
# Tailwind CSS 재구성
npx tailwindcss -i ./app/globals.css -o ./app/output.css
```

### TypeScript 오류
```bash
# TypeScript 검증
npx tsc --noEmit

# 타입 오류가 있으면 자세한 정보 표시
```
