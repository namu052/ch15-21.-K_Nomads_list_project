# KNC 프로젝트 단계별 개선 계획 SPEC

## 📋 전체 로드맵

```
Phase 1: 구조 정리 및 데이터 구조화
    ↓
Phase 2: 좋아요/싫어요 기능 (상태 관리)
    ↓
Phase 3: 카드 UI 개선 및 필터 정보 표시
    ↓
Phase 4: 필터 기능 및 정렬 기능 완성
```

---

## ✅ PHASE 1: 구조 정리 및 데이터 구조화

### 📌 개요
불필요한 페이지와 내비게이션을 제거하고, 도시 데이터에 필터 정보(예산, 지역, 환경, 계절)를 추가하여 향후 Phase의 기초를 마련합니다.

### 🔧 수정/개선 기능

- [ ] 불필요한 페이지 삭제
  - [ ] `/city` 동적 라우트 삭제 (아직 구현 예정이면)
  - [ ] `/explore` 페이지 삭제 (아직 구현 예정이면)
  - [ ] `/auth` 인증페이지 외 다른 라우트 확인 및 정리

- [ ] Header 네비게이션 정리
  - [ ] 인증(로그인/회원가입) 관련 버튼만 유지
  - [ ] 프로필 관련 버튼만 유지
  - [ ] 다른 페이지로 이동하는 버튼 모두 제거 (Home, About, Contact 등)
  - [ ] 로고는 홈페이지로 가는 링크로만 유지

- [ ] 홈페이지 섹션 조정 (현재 7개 섹션)
  - [ ] StatsSection 유지 여부 검토
  - [ ] FeaturesSection 유지 여부 검토
  - [ ] TestimonialsSection 유지 여부 검토
  - [ ] CtaSection 유지 여부 검토
  - [ ] 필요시 정리 (도시 리스트에만 집중)

- [ ] 도시 데이터 구조 개선
  - [ ] 기존 fields 확인: name, emoji, rating, cost, internet, nomads
  - [ ] 신규 필드 추가:
    - [ ] `budget`: "100만원" | "100~200만원" | "200만원"
    - [ ] `region`: "전체" | "수도권" | "경상도" | "전라도" | "강원도" | "제주도" | "충청도"
    - [ ] `environment`: string[] (예: "자연친화", "도심선호", "카페작업", "코워킹 필수" 중 최소 1개)
    - [ ] `bestSeason`: "봄" | "여름" | "가을" | "겨울"
    - [ ] `likes`: number (초기값)
    - [ ] `dislikes`: number (초기값)
  - [ ] 각 도시별로 최소 하나의 필터 요소 추가

### ✅ 검증/확인 기능

- [ ] 네비게이션 확인
  - [ ] Header에서 인증/프로필 버튼만 보이는지 확인
  - [ ] 로고 클릭 → 홈페이지 이동 확인
  - [ ] 불필요한 링크 모두 제거됨 확인

- [ ] 홈페이지 레이아웃 확인
  - [ ] 필요한 섹션만 렌더링되는지 확인
  - [ ] 레이아웃이 깨지지 않는지 확인
  - [ ] 반응형 디자인 정상 작동 확인

- [ ] 도시 데이터 출력 확인
  - [ ] 모든 도시가 정상적으로 로드됨 확인
  - [ ] 새로운 필드들이 객체에 포함되어 있는지 확인
  - [ ] 각 도시별 필터 정보가 다양하게 구성되었는지 확인

## ✅ PHASE 2: 좋아요/싫어요 기능 구현

### 📌 개요
별점(rating) 시스템을 제거하고, 좋아요/싫어요 버튼을 추가합니다. 클릭 시 상태(토글)가 변경되고, 아이콘 색상이 변하며, 카운트가 업데이트됩니다.

### 🔧 수정/개선 기능

- [ ] 별점 UI 제거
  - [ ] FeaturedCitiesSection에서 rating 필드 표시 부분 제거
  - [ ] 별점 아이콘(⭐) 제거

- [ ] 좋아요/싫어요 버튼 추가
  - [ ] 새로운 컴포넌트 또는 기존 카드에 버튼 추가
  - [ ] 버튼 구조: `👍 좋아요 (숫자)` / `👎 싫어요 (숫자)`
  - [ ] 버튼이 나란히 배치 (가로 정렬)

- [ ] 상태 관리 (Client Component 전환)
  - [ ] FeaturedCitiesSection을 'use client' 컴포넌트로 변경
  - [ ] 각 도시별 좋아요/싫어요 상태 관리
  - [ ] 사용자가 한 도시에서 좋아요 또는 싫어요만 선택 가능 (동시 선택 불가)
    - [ ] 좋아요 선택 후 싫어요 클릭 → 좋아요 취소, 싫어요 활성화
    - [ ] 싫어요 선택 후 좋아요 클릭 → 싫어요 취소, 좋아요 활성화

- [ ] 아이콘 색상 변경 로직
  - [ ] 활성 상태: 색상 변경 (예: 👍 → 파란색, 👎 → 주황색)
  - [ ] 비활성 상태: 회색 또는 기본 색상
  - [ ] 전환 애니메이션 (선택 사항)

- [ ] 카운트 업데이트
  - [ ] 좋아요 클릭 시 `likes` 숫자 +1/-1
  - [ ] 싫어요 클릭 시 `dislikes` 숫자 +1/-1

### ✅ 검증/확인 기능

- [ ] 버튼 상호작용 확인
  - [ ] 좋아요 버튼 클릭 → 활성화 (색상 변경) 및 카운트 증가
  - [ ] 싫어요 버튼 클릭 → 활성화 (색상 변경) 및 카운트 증가
  - [ ] 같은 버튼 재클릭 → 비활성화 및 카운트 감소

- [ ] 토글 로직 확인
  - [ ] 좋아요 활성 상태에서 싫어요 클릭 → 좋아요 비활성화, 싫어요 활성화
  - [ ] 싫어요 활성 상태에서 좋아요 클릭 → 싫어요 비활성화, 좋아요 활성화
  - [ ] 상태 전환 시 카운트 정확히 변경됨 확인

- [ ] UI 레이아웃 확인
  - [ ] 버튼이 카드 내 적절한 위치에 배치됨
  - [ ] 반응형 디자인에서 버튼이 깨지지 않음
  - [ ] 색상 대비가 충분함 (접근성)

- [ ] 데이터 일관성 확인
  - [ ] 페이지 새로고침 후에도 상태 유지 (localStorage 사용 가능)
  - [ ] 다른 도시 카드 상태가 서로 영향을 주지 않음

## ✅ PHASE 3: 카드 UI 개선 및 필터 정보 표시

### 📌 개요
카드 레이아웃을 변경하여 필터 정보(예산, 지역, 환경, 계절)를 Key-Value 형식으로 시각적으로 표시합니다. "자세히 보기" 버튼을 제거합니다.

### 🔧 수정/개선 기능

- [x] 카드 레이아웃 재구성
  - [x] 도시명 + 이모지 (헤더)
  - [x] 필터 정보 섹션 (Key-Value 형식)
  - [x] 상세 정보 섹션 (월 생활비, 인터넷, 노마드 수)
  - [x] 좋아요/싫어요 버튼 섹션

- [x] 필터 정보 표시 (Key-Value)
  - [x] 형식 예:
    ```
    🏷️ 예산: 100~200만원
    📍 지역: 수도권
    🌿 환경: 자연친화, 카페작업
    🌸 최고계절: 봄
    ```
  - [x] Key-Value 형식으로 표시 완료
  - [x] 각 필터별 아이콘 사용하여 구분 용이하게

- [x] "자세히 보기" 버튼 제거
  - [x] 기존 코드에 버튼 없음 (확인)
  - [x] 카드 전체에 호버 효과 유지

- [x] 카드 스타일 조정
  - [x] 필터 정보 섹션 추가로 인한 레이아웃 확장
  - [x] border-t, border-b로 필터 정보 섹션 시각적 구분
  - [x] py-3로 필터 정보 섹션 padding 조정

### ✅ 검증/확인 기능

- [x] 필터 정보 표시 확인
  - [x] 모든 도시에서 4가지 필터(예산, 지역, 환경, 계절) 모두 표시됨
  - [x] 텍스트가 명확하고 읽기 쉬움
  - [x] 아이콘이 적절히 표시됨

- [x] 카드 레이아웃 확인
  - [x] 카드 높이가 자연스럽고, 정보가 잘 정렬됨
  - [x] 기존 반응형 디자인 유지 (4열 → 2열 → 1열)
  - [x] 카드 간 간격이 일정함 (gap-6)

- [x] "자세히 보기" 버튼 제거 확인
  - [x] 카드 하단에 링크가 없음
  - [x] 카드가 호버 효과 유지 (transition-shadow)

- [x] 시각적 일관성 확인
  - [x] 색상 스키마 유지 (text-text-main, text-text-sub)
  - [x] 폰트 크기/무게 계층적 (h3: text-2xl, details: text-sm, filters: text-xs)
  - [x] 필터 정보와 나머지 정보 간 border로 시각적 구분 명확

## ✅ PHASE 4: 필터 기능 및 정렬 기능 완성

### 📌 개요
필터 UI를 구현하여 사용자가 예산, 지역, 환경, 계절로 도시를 필터링할 수 있게 하고, 좋아요 순서로 정렬합니다. 섹션 제목을 "도시 리스트"로 변경합니다.

### 🔧 수정/개선 기능

- [x] 섹션 제목 변경
  - [x] "🏙️ 도시 리스트"로 변경 완료
  - [x] 이모지 적용됨

- [x] 필터 UI 구현
  - [x] 필터 바를 섹션 헤더 아래에 배치
  - [x] 4개 필터 모두 구현 (예산, 지역, 환경, 계절)
  - [x] 각 필터별 UI 타입
    - [x] 예산: 드롭다운 (4개 옵션)
    - [x] 지역: 드롭다운 (7개 옵션)
    - [x] 환경: 멀티셀렉트 (체크박스, 4개 옵션)
    - [x] 계절: 드롭다운 (5개 옵션)

- [x] 필터링 로직

- [x] 정렬 기능
  - [x] "좋아요 순" 정렬 (기본값, 유일한 정렬 옵션)

- [x] 리셋/초기화

  - [x] 결과 없음 상태 처리 (Empty State UI)
